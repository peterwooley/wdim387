Using source control management software

  • Revision control (or version control, source control, source control management, source code management) is the management of changes in files.
  • A Typical revision control system will allow you to compare, restore, and merge various files.
  • Revision control systems typically store changes in files as deltas—the difference between two file states.
  • The simplest form of revision control is storing backups of your files as you make changes (e.g., index_01.html, index_02.html, index_03.html, etc.)

Types of revision control

  • Client-Server
    • All changes to code exist in a server-side repository.
    • Changes are commit'd to the server through clients.
    • Client repositories must be update'd from the server.
    • If the same file is modified by two users, upon commit, the file is merged.
  • Distributed
    • All changes to code exist in a client-side repository.
    • Changes are commit'd to the local repository, and push'd to other repositories.
    • Client repositories can pull changes from other repositories.
    • If the same file is modified by two users, upon push or pull, the file is merged.

Writing & generating API documentation

  • Carefully documenting your code, in a standard way, is extremely helpful.
  • JSDoc gives that standard and jsdoc-toolkit is the tool that'll make your documentation process simple.
  • A bit of JSDoc documentation to admire: 
    /**
 * Creates a SearchApp that adds live searching to a supplied table of data.
 * @class Searches table and manages visibilty of applicable search results.
 * @version 1.1
 *
 * @property {Object} self Internal reference to <code>this</code>.
 * @property {Object} root jQuery DOM Reference to root node for instance
 *
 * @param {Object} $element The root jQuery DOM reference, containing the search field and table of data.
 */
function SearchApp($element) {
	var self = this;
	this.root = $element;
	
	/**
	 * Searches through all table data for the supplied search term, hiding all rows that do not match.
	 * @returns {null} Does not return a value.
	 */
	this.search = function(event) {
		$('tbody tr', self.root.children('.list')).show().not(":contains(" + $(this).val() + ")").hide();
	}
	
	$('input', this.root).keyup(this.search);
	$('[type=submit]', this.root).hide();
}
