Firebug Tricks: Magic console.log( )

  1. Open the Script tab and find the line where you'd like to add the console.log().
  2. Right-click to the left of the line and enter this code: console.log(expession) && 0
  3. Run the code and watch the expression be logged in the Console Tab.
  4. This utilizies conditional breakpoints, where a breakpoint will only break if the expression is true. With the && 0, the expression will always return false, regardless of what console.log returns.

Firebug Trick: Collapsible Console Group

  1. Where you want to use console.log(), console.error(), etc., just prepend console.group() and append console.groupEnd().
  2. This is useful when you want to group a large number of messages without putting them into a single console.log message.
  3. You can also use console.groupCollapsed(), which automatically collapses the group.

Firebug Trick: Timer without a Date( )!

  • Where you want to start a timer, use console.time(nameOfTimer). You can start as many timers are you'd like using unique names.
  • To end a timer and write the elapsed time to the console, just call console.timeEnd(nameOfTimer).

Issue Tracking

Issue Tracking is a necessity for nearly any project.

Picking an issue tracker should consider:

  • Accesibility (Notebook, .txt, hosted, etc.)
  • Scalability (Components, tags, search, assignees)
  • SCM Hooks (Commit message hooks, issue linking)
  • User Experience (More important than you'd think)

GitHub Issues 2.0

Included for every repo, including wdim387.

  • Hosted on GitHub.
  • Search, sort, and filter, including labels (e.g. tags)
  • Close issues from commit messages.
  • Fast, Ajax interface, designed for getting through issues quickly.
  • Issue voting, so users can set the priority.

Bugzilla

Mozilla & NASA use it!

  • Self-Hosted, runs on Perl, Open Source.
  • Advanced query tool that can remember your searches, but is quite complicated.
  • Extremely scalable, both through server performance and number of hosted projects.
  • Comprehensive permissions system.
  • On the Difficult-To-Use-O-Meter, it rates a 9/10.

Trac

  • Self-Hosted, runs on Python, Open Source.
  • Extremely customizable, Wiki-like interface with support for advanced searches.
  • Includes Issue Tracking as well Project Management using Issues as Features.
  • Manage existing issues from commit messages.
  • In my experience, it was a bit complicated to setup, but the integrated Project Management was worth it!

Coding Standards

  • Every language has a set of coding standards (or multiple sets) that encourage unambiguous code.
  • Coding style is an extremely personal topic for some developers.
  • Agreeing on a project's style should be done as early as possible.
  • Do not rewrite someone else's code just to change its style.

JavaScript Coding Style

In the realm of JavaScript, Douglas Crockford keeps a list of good conventions, the key points being:

  • Declare functions & variables before they are used.
  • The with statement should not be used.
  • eval is evil and should not be used.

The canonical style guide is Google's JavaScript Style Guide.

PHP Coding Style

CodeIgniter has its own Style Guide, which contain a few great ideas like:

  • Always use the full-form PHP tags (<?php).
  • For files that contain only PHP code, the closing tag (?>) is never permitted.
  • TRUE, FALSE, and NULL should always be upper-case.
  • If a class name is comprised of more than one word, the first letter of each new word must be prepended by an underscore and not CamelCased. (e.g. Super_class)
