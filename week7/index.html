Coding Standards

  • Every language typically has a set of coding standards (or multiple sets) that encourage unambiguous code.
  • In the realm of JavaScript, Douglas Crockford keeps a list of good conventions, the key points being:
    • Declare functions & variables before they are used.
    • The with statement should not be used.
    • eval is evil and should not be used.
  • CodeIgniter has its own Style Guide, which contain a few great ideas like:
    • Always use the full-form PHP tags (<?php).
    • For files that contain only PHP code, the closing tag (?>) is never permitted.
    • TRUE, FALSE, and NULL should always be upper-case.
    • If a class name is comprised of more than one word, the first letter of each new word must be prepended by an underscore and not CamelCased. (e.g. Super_class)

User Stories

  • Planning features for a project can be made easier by using User Stories.
  • User Stories describe how a user would use a feature of a given app.
  • When writing a story, start with "A user can...". For example:
    • A user can register for the application with their email address.
    • A user can register for the application using Twitter OAuth.
    • A user can login using a login form, or through Twitter OAuth, on the same page.
    • A user can view the number of people they are following on every page of the app.

Engineering tasks & Acceptance Tests

  • For each User Story, several engineering stories and acceptance tests are created.
  • An engineering task describes what the developer will need to do to fulfill the user story.
  • An acceptance test is a simple test that can be performed when presenting the user story to prove it works.
  • Using the "User can register for the application with their email address" story, the tasks and tests might look like this:
    • Task: Create database tables for user accounts.
    • Task: Create registration form with necessary fields (email, password, confirmation, terms, etc.).
    • Task: Write the PHP to send the form, validate the form, and add the user or reject with an error.
    • Test: Go to registration page, fill out form fields and get confirmation of account creation.
    • Test: Attempt to register with invalid information and get correct error messages.
